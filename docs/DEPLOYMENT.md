# TrustLance Local Deployment & Webhook Setup

To test the end-to-end payment capture and escrow state transitions locally, Razorpay webhook events must be routed successfully to your local Next.js server.

## Webhook Setup for Local Development

Since Razorpay's servers are on the public internet, they cannot directly communicate with `localhost`. You must set up a public HTTPS tunnel using a tool like **ngrok**.

### 1. Requirements
- **ngrok** must be running during your development/testing session.
- The active ngrok URL must be registered in your Razorpay Dashboard webhook settings before testing any real payment flow locally.

### 2. Steps to Configure ngrok Tunnel
1. Start your local Next.js server:
   ```bash
   npm run dev
   ```
2. Start the ngrok HTTP tunnel on port 3000:
   ```bash
   ngrok http 3000
   ```
3. Copy the public forwarding HTTPS URL generated by ngrok (e.g., `https://xxxx-xxxx-xxxx.ngrok-free.dev`).
4. Log in to the **Razorpay Dashboard**:
   - Go to **Settings > Webhooks**.
   - Create a new webhook or update the existing one.
   - Set the **Webhook URL** to `{your-ngrok-url}/api/webhooks/razorpay`.
   - Set the **Secret** to the value matching `RAZORPAY_WEBHOOK_SECRET` in your local `.env` file (e.g., `trustlance_webhook_9x7F@2KpLmQ2026`).
   - Select the active event: `payment.captured`.
   - Save the settings.

> [!WARNING]
> If ngrok is restarted, a new public URL will be generated. You must update the Webhook URL in the Razorpay Dashboard with the new URL to avoid webhook delivery failures.

## Auto-Release Cron & Manual Trigger

TrustLance incorporates an auto-release cron mechanism to resolve stalled reviews automatically after a grace period.

### 1. Behavior & Settings
- **Grace Period**: 5 days (120 hours) after work is submitted (project reaches `UNDER_REVIEW`).
- **24-Hour Advance Warning**: Clients receive a warning email 24 hours prior to auto-release, giving them one last chance to approve, request changes, or raise a dispute.
- **System Actor**: Transitions executed by the cron are attributed in the audit logs to `SYSTEM_AUTO_RELEASE`, rendering them clearly distinguishable from manual approvals.

### 2. Manual Execution Trigger
During local development, instead of scheduling a node-cron server task, you can manually trigger the auto-release logic at any time:
- Send a authenticated `POST` request from an **Admin** session to: `/api/admin/run-auto-release`.
- Response format:
  ```json
  {
    "success": true,
    "released": 1,
    "warned": 0
  }
  ```
- Alternatively, go to the Admin Overview Dashboard at `/admin/overview` and click the **"Trigger Auto-Release"** button.
- For production environments, Vercel Cron or a separate cron runner can be scheduled to hit the `/api/admin/run-auto-release` endpoint on a daily schedule.

## Vercel Production Deployment Checklist

Use this checklist to deploy TrustLance to Vercel in production.

### 1. Environment Variable Setup
Configure the following variables in the **Vercel Project Settings > Environment Variables**:

| Variable | Description / Value |
| --- | --- |
| `DATABASE_URL` | Neon/Supabase PostgreSQL connection string (including connection pooling parameters). |
| `NEXTAUTH_URL` | Your production application base URL (e.g. `https://trustlance.vercel.app`). |
| `NEXTAUTH_SECRET` | A secure random string (generate using `openssl rand -base64 32`). |
| `GOOGLE_CLIENT_ID` | Production Client ID from Google Cloud Console. |
| `GOOGLE_CLIENT_SECRET` | Production Client Secret from Google Cloud Console. |
| `RAZORPAY_KEY_ID` | Production key ID from Razorpay Dashboard (API Keys). |
| `RAZORPAY_KEY_SECRET` | Production key secret from Razorpay Dashboard. |
| `RAZORPAY_WEBHOOK_SECRET` | Secure string configured in Razorpay Webhooks (sent in payload verification). |
| `AWS_ACCESS_KEY_ID` | AWS IAM user access key with S3 put/get object permissions. |
| `AWS_SECRET_ACCESS_KEY` | AWS IAM user secret access key. |
| `AWS_REGION` | S3 bucket region (e.g., `ap-south-1`). |
| `AWS_S3_BUCKET_NAME` | Name of your production S3 bucket. |
| `RESEND_API_KEY` | Production API key from Resend. |
| `CRON_SECRET` | Secure random token generated by Vercel for scheduling crons. |

### 2. Google OAuth Configuration
- Go to [Google Cloud Console](https://console.cloud.google.com/).
- Navigate to **APIs & Services > Credentials**.
- Edit your OAuth 2.0 Client ID.
- Add your production URL to **Authorized JavaScript origins** (e.g. `https://trustlance.vercel.app`).
- Add the callback URL to **Authorized redirect URIs**:
  - `{production-url}/api/auth/callback/google`

### 3. Razorpay Webhook Configuration
- Log in to the [Razorpay Dashboard](https://dashboard.razorpay.com/).
- Go to **Account & Settings > Webhooks**.
- Add a new webhook with URL:
  - `{production-url}/api/webhooks/razorpay`
- Input your `RAZORPAY_WEBHOOK_SECRET`.
- Subscribe to the `payment.captured` event.

### 4. Database Setup & Migrations
- In your Vercel deployment pipeline, configure the build command to run migrations before generating the client:
  ```json
  "build": "prisma generate && prisma migrate deploy && next build"
  ```
- Or run migrations manually from a production-capable local console.

### 5. Cron Job Scheduling
- Vercel automatically detects the `vercel.json` crons entry and schedules a daily trigger for `/api/admin/run-auto-release`.
- Verify the status of the cron under the **Vercel Project Dashboard > Crons** tab after deployment.
